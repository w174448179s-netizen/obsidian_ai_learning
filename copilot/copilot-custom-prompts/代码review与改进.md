对用户提供的AI应用代码进行专业review并给出改进建议。

## 执行步骤
1. 分析代码的业务逻辑和架构合理性
2. 检查代码规范性和最佳实践
3. 识别潜在问题和优化点
4. 结合[[MOC-架构蓝图]]提出架构层面改进
5. 提供改进后的代码示例

## 输出格式
### 📋 代码分析
代码功能概述和整体评价

### ✅ 优点
- 优点1
- 优点2

### ❌ 问题与改进
| 问题类型 | 问题描述 | 改进建议 |
|---------|---------|---------|
| 架构设计 | 描述 | 建议 |
| 代码规范 | 描述 | 建议 |
| 性能优化 | 描述 | 建议 |
| 安全隐患 | 描述 | 建议 |

### 🔧 改进后代码
```java
// 优化后的代码
```

### 💡 架构建议
结合架构蓝图提出的整体改进思路

## 使用方式
用户粘贴代码后，点击此提示词进行review。

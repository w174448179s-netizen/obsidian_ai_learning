# 数据库表结构

> 最后更新：2026-07-19

## 核心数据表

### 用户表 (users)

- id: 主键
- openid: 微信openid（唯一）
- unionid: 微信unionid
- nickname: 昵称
- avatar_url: 头像
- phone: 手机号
- gender: 性别
- birthday: 生日
- points: 积分
- level: 会员等级
- balance: 储值余额
- status: 状态（正常/禁用）
- created_at / updated_at

### 商品分类表 (categories)

- id: 主键
- name: 分类名称
- parent_id: 父分类ID（支持多级）
- icon: 分类图标
- sort_order: 排序
- status: 状态
- created_at / updated_at

### 商品表 (products)

- id: 主键
- name: 商品名称
- category_id: 分类ID
- price: 现价
- original_price: 原价
- cover_image: 封面图
- images: 图片列表(JSON)
- description: 详细描述
- flower_meaning: 花语
- flower_materials: 花材说明
- stock: 库存
- sales_count: 销量
- status: 状态（上架/下架）
- sort_order: 排序
- created_at / updated_at

### 商品规格表 (product_specs)

- id: 主键
- product_id: 商品ID
- name: 规格名称（如：小号/中号/大号）
- price: 规格价格
- stock: 规格库存
- created_at / updated_at

### 订单表 (orders)

- id: 主键
- order_no: 订单编号
- user_id: 用户ID
- total_amount: 订单总额
- pay_amount: 实付金额
- status: 状态（待支付/已支付/制作中/配送中/已完成/已取消/已退款）
- receiver_name: 收货人
- receiver_phone: 收货电话
- receiver_address: 收货地址
- delivery_time: 配送时间
- remark: 备注
- pay_time: 支付时间
- delivery_time_actual: 实际配送时间
- complete_time: 完成时间
- created_at / updated_at

### 订单明细表 (order_items)

- id: 主键
- order_id: 订单ID
- product_id: 商品ID
- spec_id: 规格ID
- product_name: 商品名称（快照）
- spec_name: 规格名称（快照）
- price: 单价（快照）
- quantity: 数量
- subtotal: 小计
- created_at

### 购物车表 (cart_items)

- id: 主键
- user_id: 用户ID
- product_id: 商品ID
- spec_id: 规格ID
- quantity: 数量
- created_at / updated_at

### 收藏表 (favorites)

- id: 主键
- user_id: 用户ID
- product_id: 商品ID
- created_at

### 评价表 (reviews)

- id: 主键
- user_id: 用户ID
- product_id: 商品ID
- order_id: 订单ID
- rating: 评分（1-5）
- content: 评价内容
- images: 评价图片(JSON)
- merchant_reply: 商家回复
- created_at / updated_at

### 地址表 (addresses)

- id: 主键
- user_id: 用户ID
- receiver_name: 收货人
- receiver_phone: 收货电话
- province / city / district: 省市区
- detail: 详细地址
- is_default: 是否默认
- created_at / updated_at

### 优惠券表 (coupons)

- id: 主键
- name: 优惠券名称
- type: 类型（满减/折扣/无门槛）
- discount_value: 优惠值
- min_amount: 最低使用金额
- valid_start / valid_end: 有效期
- total_count: 发放总量
- used_count: 已使用数量
- status: 状态
- created_at

### 用户优惠券表 (user_coupons)

- id: 主键
- user_id: 用户ID
- coupon_id: 优惠券ID
- order_id: 使用订单ID（关联已使用的订单）
- status: 状态（未使用/已使用/已过期）
- received_at: 领取时间
- used_at: 使用时间

## 设计要点

- 订单明细中保存商品名称和价格的快照，防止商品信息变更影响历史订单
- 商品支持多规格（花束大小/包装等），规格独立管理库存和价格
- 用户地址独立表管理，支持多地址和默认地址
- 优惠券采用发放+领取+使用三段式

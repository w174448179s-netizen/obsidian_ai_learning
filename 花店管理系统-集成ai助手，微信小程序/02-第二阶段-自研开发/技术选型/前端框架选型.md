# 前端框架选型

> 最后更新：2026-07-19

## 三种方案对比

| 维度 | 原生微信小程序 | uni-app (Vue 3) | Taro (React) |
|------|---------------|-----------------|--------------|
| 技术栈 | WXML/WXSS/JS/TS | Vue 3 + UniCloud | React + TypeScript |
| 性能 | 最优，无中间层损耗 | 编译器优化后差距缩小 | 编译时优化，接近原生 |
| 开发效率 | 低 | 高 | 中高 |
| 多端覆盖 | 仅微信 | 微信/支付宝/抖音/鸿蒙/App/H5 | 同上 |
| 学习曲线 | 中 | 低 | 中高 |
| 插件生态 | 官方组件库 | 庞大插件市场 | 大厂背书，类型安全 |
| AI集成 | 直接调用微信AI Kit | uni-ai统一接口层 | 高阶AI组件封装 |
| 鸿蒙适配 | 需ArkTS独立开发 | 插件形式支持 | 源码级转换较精准 |

## 推荐：uni-app (Vue 3)

### 理由

- 花店电商属于中小型项目，核心需求是快速上线和多端覆盖
- Vue 3 生态丰富，学习成本低
- 插件市场有大量现成电商组件
- uniCloud 提供Serverless后端，前端工程师可一人搞定前后端
- 后续如需覆盖支付宝/抖音小程序，无需重写代码

### 适用条件

- 团队熟悉Vue或愿意学习Vue
- 需要多端覆盖（不止微信）
- 追求开发效率而非极致性能

### 不适用场景

- 团队深度使用React（此时选Taro更合适）
- 对性能有极致要求（如直播卖花、AR花束展示）
- 需要第一时间使用微信最新API

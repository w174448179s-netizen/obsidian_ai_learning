# 系统架构设计

> 最后更新：2026-07-19

## 整体架构

- 小程序前端层（uni-app）
- API网关层（Nginx反向代理）
- 业务服务层（Node.js + Koa）
- 数据存储层（MySQL + Redis + OSS）
- 第三方服务层（微信支付/短信/物流推送）
- AI服务层（第三阶段新增）

## 前端架构

- 框架：uni-app (Vue 3)
- 状态管理：Pinia
- 网络请求：封装uni.request + 拦截器
- 组件库：uView UI / uni-ui
- 分包策略：主包（首页/分类/购物车/我的）+ 分包（订单/活动/AI对话）

## 后端架构

- 框架：Koa 2.x + 中间件模式
- 分层：Controller → Service → Model
- API设计：RESTful风格
- 鉴权：JWT Token + 微信登录session
- 错误处理：统一错误码 + 全局异常捕获
- 日志：请求日志 + 业务日志 + 错误日志

## 数据层

- MySQL：业务数据主存储
- Redis：会话缓存、商品缓存、库存缓存、分布式锁
- OSS：花束图片、用户头像、评价图片

## 部署架构

- 云服务器：2核4G（腾讯云/阿里云）
- Nginx：反向代理 + HTTPS + 静态资源
- PM2：Node.js进程管理 + 自动重启
- 域名：已备案域名 + Let's Encrypt SSL

## 后台管理系统

- 框架：Vue 3 + Element Plus
- 功能：商品管理、订单管理、会员管理、数据统计、营销活动、内容管理
- 部署：与API同服务器，Nginx区分路由

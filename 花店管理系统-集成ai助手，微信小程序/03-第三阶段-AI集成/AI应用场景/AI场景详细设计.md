# AI场景详细设计

> 最后更新：2026-07-19

## 场景一：智能客服（优先级：高）

### 功能描述

AI客服自动回答用户常见问题，降低人工客服成本

### 能力范围

- 营业时间、门店地址查询
- 配送范围和配送时间咨询
- 花材、价格、规格咨询
- 订单状态查询
- 售后政策说明
- 复杂问题转接人工客服

### 技术实现

- 使用RAG从花店知识库检索相关内容
- 后端维护FAQ知识库（配送规则、售后政策、常见花语、养护指南）
- 用户输入 → 文本安全检测 → RAG检索 → 大模型生成回答
- 支持多轮对话，通过会话ID维护上下文

### 交互方式

- 小程序内嵌AI对话入口（悬浮按钮）
- 支持文字输入
- 首次进入显示快捷问题标签（如"配送范围""花语查询""订单查询"）

## 场景二：花语解读（优先级：高）

### 功能描述

根据用户输入的花材组合，解释不同花材的花语和寓意

### 能力范围

- 单花材花语查询（如"玫瑰代表什么"）
- 花束组合花语解读（如"玫瑰+百合是什么意思"）
- 送礼场景推荐（如"送给妈妈选什么花"）
- 节日花束推荐（如"母亲节送什么"）

### 技术实现

- 花材知识库（花名、花语、寓意、适合场景、价格区间）
- RAG检索花材知识 → 大模型生成自然语言解读
- 可结合Function Calling查询实时库存和价格

### 示例对话

```
用户：玫瑰加百合适合送给谁？
AI：玫瑰代表爱情，百合象征纯洁与祝福。玫瑰+百合的组合
    适合送给爱人（表达爱意与祝福）或新婚夫妇（爱情与美满）。
    目前店内有"浪漫时光"花束（3支玫瑰+2支百合+尤加利叶），
    售价198元，点击查看详情→
```

## 场景三：花束DIY推荐（优先级：中）

### 功能描述

根据用户预算、送花对象、使用场景，AI推荐个性化花束方案

### 能力范围

- 按预算推荐（如"200元左右送女朋友"）
- 按场景推荐（如"开业花篮""婚礼手捧花"）
- 按花材偏好推荐（如"不要菊花，喜欢暖色调"）
- 根据已有花材搭配推荐

### 技术实现

- Function Calling调用后端工具：
  - `recommend_bouquet(budget, occasion, recipient, preferences)`
- 后端根据参数查询可用商品和库存
- 大模型将推荐结果转化为自然语言
- 推荐结果带商品链接，支持一键加购

### 注意事项

- 推荐结果必须与实际库存和价格匹配
- AI只生成推荐文案，不直接创建订单
- 后端做二次校验，避免推荐不存在的商品

## 场景四：养护指南（优先级：中）

### 功能描述

根据花材类型给出专业养护建议

### 能力范围

- 按花材给出养护方法（换水频率、修剪方法、温度控制）
- 按季节给出养护建议
- 鲜切花 vs 干花 vs 绿植的不同养护方法
- 花材寿命预估

### 技术实现

- 养护知识库（按花材分类的养护文档）
- RAG检索 → 大模型生成个性化养护建议

## 场景五：节日提醒（优先级：中）

### 功能描述

自动识别即将到来的节日，推送预订提醒和推荐

### 能力范围

- 节日日历（情人节、母亲节、七夕、教师节、圣诞节等）
- 提前3-7天推送提醒
- 根据节日推荐对应花束
- 结合会员生日、纪念日推送个性化提醒

### 技术实现

- 后端定时任务扫描节日日历
- 结合用户画像（会员等级、历史订单、生日）生成个性化推荐
- 通过微信模板消息/订阅消息推送

## 场景六：销售数据分析（优先级：低）

### 功能描述

为花店老板提供AI驱动的经营分析

### 能力范围

- 生成销售日报、周报、月报
- 节日前后订单变化分析
- 热销/滞销花材识别
- 补货和促销建议

### 技术实现

- 后端查询业务数据库（订单、商品、会员）
- 将数据摘要传递给大模型
- 大模型生成分析报告和建议
- 仅在后台管理系统展示，不暴露给C端用户

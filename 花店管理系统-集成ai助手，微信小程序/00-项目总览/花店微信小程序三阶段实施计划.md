> 项目目标：以最低成本快速验证花店线上业务，逐步过渡到自研系统，最终集成AI Agent实现智能化运营

## 第一阶段：SaaS快速上线（1-2周）

**目标：** 以最低费用（约700元/年起）快速上线花店微信小程序，验证线上业务模式

**核心策略：** 使用SaaS平台模板，零代码/低代码快速搭建

### 推荐方案：乔拓云标准版

- 费用：1399元/年（买2送2活动，折算699.5元/年）
- 包含：模板、服务器、域名、备案、售后维护
- 支持功能：商品管理、订单管理、会员体系、拼团/秒杀/优惠券/分销

### 备选方案对比

| 平台 | 年费 | 优势 | 适合场景 |
|------|------|------|----------|
| 乔拓云 | 699.5元（折算） | 性价比最高、功能较全 | 预算最低 |
| 固乔建站 | 700元 | 入门门槛低 | 轻量化展示型 |
| 得有店 | 1999元 | 功能较完善 | 中小商家 |
| 有赞 | 6800元 | 私域电商深耕 | 品牌商家 |
| 微盟 | 6800元 | 全渠道数字化 | 连锁门店 |

### 上线清单

详见 [[01-第一阶段-快速上线/上线清单/SaaS上线操作清单]]

- [ ] 注册微信小程序账号（300元/年认证费）
- [ ] 注册微信支付商户号
- [ ] 选择SaaS平台并购买套餐
- [ ] 选择花店行业模板
- [ ] 录入商品信息（花束图片、价格、描述）
- [ ] 配置配送范围和规则
- [ ] 配置支付方式
- [ ] 测试完整下单流程
- [ ] 提交微信审核并发布

### 第一阶段功能范围

- 商品展示（花束分类、图片、价格、花语描述）
- 在线下单（规格选择、配送时间、留言备注）
- 微信支付
- 基础订单管理
- 基础会员功能

### 预算

| 项目 | 费用 |
|------|------|
| 微信认证 | 300元/年 |
| SaaS平台 | 700元/年（乔拓云折算） |
| **首年合计** | **约1000元** |

## 第二阶段：自研开发（2-3个月）

**目标：** 基于第一阶段积累的用户和业务数据，自研一套功能完善的电商小程序

**触发条件：** 第一阶段月订单量稳定增长、SaaS平台功能无法满足定制需求

### 技术选型

详见 [[02-第二阶段-自研开发/技术选型/前端框架选型]] 和 [[02-第二阶段-自研开发/技术选型/后端技术选型]]

| 维度   | 推荐方案            | 理由                  |
| ---- | --------------- | ------------------- |
| 前端框架 | uni-app (Vue 3) | 快速迭代、多端覆盖、插件生态丰富    |
| 后端语言 | Node.js + Koa   | 与前端技术栈统一，全栈工程师可独立完成 |
| 数据库  | MySQL           | 关系型数据库，适合电商场景       |
| 部署方案 | 自建云服务器          | 灵活性高、成本可控、数据自主      |
| 缓存   | Redis           | 高频数据缓存（商品、分类、用户会话）  |
| 文件存储 | 对象存储(OSS)       | 花束图片、用户头像等静态资源      |

- 多端适配（微信+支付宝+抖音小程序，H5）
- 完整会员体系（等级、积分、储值、专属价）

## 第三阶段：AI Agent集成（1-2个月）

**目标：** 在自研系统基础上集成AI Agent，实现智能客服、花语推荐、花束定制、养护指南、销售分析等智能化能力

**触发条件：** 自研系统稳定运行、积累了足够的商品和用户数据

### AI应用场景

详见 [[03-第三阶段-AI集成/AI应用场景/AI场景详细设计]]

| 场景 | 说明 | 优先级 |
|------|------|--------|
| 智能客服 | 自动回答营业时间、配送范围、花材、价格等问题 | 高 |
| 花语解读 | 根据花材组合解释花语，推荐送礼场景 | 高 |
| 花束DIY推荐 | 根据预算、对象、场景推荐花束方案 | 中 |
| 养护指南 | 根据花材类型给出养护建议 | 中 |
| 节日提醒 | 自动推送节日预订提醒和推荐 | 中 |
| 销售数据分析 | 生成销售报告，识别热销/滞销花材 | 低 |

### 技术方案

详见 [[03-第三阶段-AI集成/技术方案/AI集成架构设计]]

### 推荐大模型

| 模型 | 输入价格 | 输出价格 | 特点 |
|------|----------|----------|------|
| DeepSeek V3 | ¥2/1M tokens | ¥8/1M tokens | 性价比高，支持Function Calling |
| 通义千问Qwen | ¥2/1M tokens | ¥6/1M tokens | 工具调用能力强 |
| 豆包2.0 | ¥0.8/1M tokens | ¥2/1M tokens | 价格最低 |
| 智谱GLM-4 | ¥5/1M tokens | ¥5/1M tokens | 中文理解能力强 |

### 核心技术

- **Function Calling / Tool Use**：花束推荐、库存查询、订单查询
- **RAG检索增强生成**：客服知识库、养护指南、节日推荐
- **知识库建设**：花材信息、养护指南、配送规则、节日指南

详见 [[03-第三阶段-AI集成/知识库建设/花店知识库规划]]

### 架构

```
小程序前端 → 后端服务 → 大模型API
                    → 向量数据库（RAG）
                    → 业务数据库
                    → 内容安全接口
```

### 预算

| 项目 | 费用 |
|------|------|
| 大模型API | 按调用量（初期可用免费额度，月费约50-500元） |
| 向量数据库 | 开源方案（ChromaDB）免费，或云服务约200-1000元/月 |
| 额外服务器资源 | 500-1500元/年（如需独立AI服务） |
| **增量成本** | **约1000-5000元/年** |

## 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| SaaS平台功能限制 | 用户体验不佳 | 第一阶段主要验证市场，不过度依赖SaaS功能 |
| 自研开发周期超期 | 上线延迟 | 采用敏捷开发，分批交付核心功能 |
| AI生成内容不准确 | 用户投诉 | 内容安全审核+后端规则二次校验+人工兜底 |
| 大模型API成本超预期 | 运营成本增加 | 缓存高频问题、限制调用频率、选择低价模型 |
| 微信审核不通过 | 无法发布 | 提前了解审核规则，准备合规材料 |

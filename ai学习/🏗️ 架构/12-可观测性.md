# 十二、可观测性 Observability

## 一句话职责
全面监控系统运行状态，快速定位问题。

## 一、Trace链路追踪

### 核心概念
- **TraceId**：全局唯一标识，贯穿整个请求链路
- **Span**：单个操作的耗时记录
- **OpenTelemetry**：统一的可观测性标准

### 保存位置
- **Jaeger**：开源分布式追踪系统
- **Tempo**：轻量级追踪系统

### Trace传递
- MDC（Mapped Diagnostic Context）
- 请求头传递
- 响应头返回

## 二、Metrics指标

### Gateway指标
- **QPS**：每秒请求数
- **延迟**：请求处理耗时
- **错误率**：失败请求比例
- **限流次数**：被限流的请求数
- **SSE连接数**：活跃的SSE连接数

### AI指标
- **Token数量**：输入/输出Token数
- **模型耗时**：模型推理耗时
- **Agent耗时**：Agent执行耗时
- **RAG耗时**：检索增强生成耗时
- **成本**：调用成本

### 存储
- **Prometheus**：时序数据库
- **Grafana**：可视化展示

## 三、Logging日志

### 日志类型
- **Access Log**：访问日志，记录所有请求
- **Error Log**：错误日志，记录异常信息
- **Trace关联**：日志与TraceId关联

### 存储
- **Loki**：轻量级日志系统
- **Elasticsearch**：全文搜索

### 日志格式

```
[traceId] [userId] [model] [path] [method] [status] [duration] [tokens]
```

## 可观测数据流

```
应用产生数据 → Micrometer采集 → Actuator暴露
            → Prometheus采集 → Grafana展示
```

## 代码片段

### TraceId Filter

```java
@Component
public class TraceIdFilter implements WebFilter {
    
    private static final String TRACE_ID_HEADER = "X-Trace-Id";
    private static final String TRACE_ID_ATTR = "traceId";
    
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, WebFilterChain chain) {
        String traceId = exchange.getRequest().getHeaders()
            .getFirst(TRACE_ID_HEADER);
        
        if (traceId == null) {
            traceId = UUID.randomUUID().toString();
        }
        
        exchange.getAttributes().put(TRACE_ID_ATTR, traceId);
        exchange.getResponse().getHeaders().add(TRACE_ID_HEADER, traceId);
        
        return chain.filter(exchange);
    }
}
```

## 设计考量

- 全链路Trace埋点，便于问题定位
- 日志与TraceId关联，快速追踪
- Metrics指标需要覆盖核心业务场景
- 响应式环境下MDC需要特殊处理

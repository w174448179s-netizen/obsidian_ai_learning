# 十五、完整请求链路

## 一句话职责
描述从客户端请求到响应返回的完整处理流程。

## 请求链路全景

### Client
- 用户请求
- 携带认证信息
- 建立连接

### Gateway
- **Authentication**：认证鉴权
- **Rate Limit**：限流检查
- **Circuit Breaker**：熔断检查
- **Routing**：路由选择
- **TraceId生成**：链路追踪

### Chat Service
- **Conversation管理**：会话管理
- 加载对话历史
- 保存对话记录

### Agent Runtime
- **意图识别**：理解用户意图
- **Tool调用**：调用外部工具
- 工作流编排

### RAG
- **Retrieval**：向量检索
- **Context构建**：拼接上下文

### Model Adapter
- **LLM调用**：调用模型API
- **Token统计**：统计Token消耗

### SSE
- **流式返回**：逐Token推送

### Observability
- **Trace**：链路追踪
- **Metrics**：指标采集
- **Logs**：日志记录

## 关键节点说明

### 认证阶段
- JWT Token验证
- 用户身份提取
- 权限校验

### 流量控制阶段
- 限流检查（QPS/并发/配额）
- 熔断检查（下游服务健康状态）
- 降级判断

### 路由阶段
- 模型选择
- 灰度路由
- 服务发现

### 业务处理阶段
- 意图识别
- 工具调用
- RAG检索
- Prompt构建

### 模型调用阶段
- 参数归一化
- API调用
- 流式处理

### 响应阶段
- SSE推送
- Token统计
- 日志记录

## 设计考量

- 全链路追踪贯穿所有模块
- 每个阶段都有异常处理和降级策略
- 响应式流式处理保证用户体验
- 可观测性覆盖关键节点

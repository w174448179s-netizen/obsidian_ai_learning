# 八、模型适配层 Model Adapter

## 一句话职责
屏蔽不同模型厂商的API差异，提供统一的模型调用接口。

## 统一模型接口

### ModelAdapter 接口

```java
public interface ModelAdapter {
    /** 非流式调用 */
    CompletableFuture<ModelResponse> chat(ModelRequest request);
    
    /** 流式调用 */
    Flux<String> streamChat(ModelRequest request);
    
    /** 路由判断：是否支持该模型类型 */
    boolean supports(ModelType modelType);
    
    /** 健康检查 */
    boolean healthCheck();
}
```

## 支持模型

### 主流模型
- **GPT**：OpenAI系列模型
- **Claude**：Anthropic系列模型
- **Qwen**：阿里云通义千问
- **DeepSeek**：深度求索

### 模型差异
- API格式差异（消息格式、参数命名）
- 参数范围差异（temperature 0-2 vs 0-1）
- 响应格式差异
- 认证方式差异

## 参数归一化

### 归一化策略
- **temperature**：统一映射到 0-1 范围
- **maxTokens**：统一单位
- **topP**：统一映射
- 其他厂商特有参数：透传

### 参数映射示例

```java
public class OpenAiAdapter implements ModelAdapter {
    private static final double TEMPERATURE_SCALE = 2.0; // GPT: 0-2
    
    @Override
    public CompletableFuture<ModelResponse> chat(ModelRequest request) {
        var openAiRequest = OpenAIRequest.builder()
            .model(request.getModel())
            .messages(mapMessages(request.getMessages()))
            .temperature(request.getTemperature() * TEMPERATURE_SCALE)
            .maxTokens(request.getMaxTokens())
            .build();
        // 调用 OpenAI SDK
    }
}
```

## 统一请求/响应模型

### ModelRequest

```java
public class ModelRequest {
    private String model;
    private List<Message> messages;
    private double temperature;
    private int maxTokens;
    private String userId;
    private String traceId;
}
```

### ModelResponse

```java
public class ModelResponse {
    private String content;
    private int inputTokens;
    private int outputTokens;
    private String finishReason;
    private String model;
}
```

## 设计考量

- 适配器模式：开闭原则，新增模型只需添加新适配器
- 参数归一化：对外统一接口，对内处理差异
- 错误处理：统一异常类型，便于上层处理

# 二、认证鉴权 Authentication

## 一句话职责
验证用户身份，管理访问凭证，保护系统安全。

## 用户身份认证

### 认证方式
- **JWT Token**：无状态认证，携带用户信息
- **OAuth2**：第三方授权，支持多种授权模式
- **SSO**：单点登录，统一身份管理
- **API Key**：简单密钥认证，适合服务间调用
- **mTLS**：双向证书认证，高安全场景

### Token体系

#### Access Token
- 短生命周期（通常15-30分钟）
- 请求接口使用
- 包含用户身份信息、角色、权限

#### Refresh Token
- 长生命周期（通常7-30天）
- 重新获取Access Token
- 通常在认证中心处理，网关不直接管理

## 网关职责

- 验证Token是否合法（签名、过期时间）
- 获取用户身份信息（从JWT Payload或认证中心）
- 注入用户上下文（放入请求Header或MDC）
- 不做业务权限判断，只做身份认证

## 后续服务职责

- 业务权限判断（基于用户角色）
- Agent权限（用户可用哪些Agent）
- 数据权限（用户可访问哪些数据）

## 鉴权链路

1. 用户登录SSO → 获取JWT Token（携带userId/role信息）
2. 请求携带JWT + X-Logic-Model-Id 进入网关
3. 网关校验JWT签名/iss/expiry（JWKS公钥缓存验签，非每轮回调SSO）
4. 网关读取role信息，Redis校验分级令牌桶限流配额
5. 合法请求转发至Agent服务

## 代码片段

### WebFilter 认证示例

```java
@Component
public class AuthFilter implements WebFilter {
    
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, WebFilterChain chain) {
        String authHeader = exchange.getRequest().getHeaders().getFirst("Authorization");
        
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            return unauthorized(exchange);
        }
        
        String token = authHeader.substring(7);
        return validateToken(token)
            .flatMap(claims -> {
                exchange.getAttributes().put("userId", claims.get("userId"));
                return chain.filter(exchange);
            })
            .onErrorResume(e -> unauthorized(exchange));
    }
}
```

## 设计考量

- API Key不能明文暴露给前端，通过网关统一管理
- JWT验签使用JWKS公钥缓存，避免每请求回调SSO
- 用户上下文在网关层注入，后续服务直接使用
- 响应式环境下MDC需要特殊处理（Spring Boot 3.2+ ObservationRegistry）

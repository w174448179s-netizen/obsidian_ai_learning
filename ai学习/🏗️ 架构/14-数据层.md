# 十四、数据层 Data Platform

## 一句话职责
管理系统所需的各类数据存储。

## MySQL

### 存储内容
- **用户数据**：用户信息、角色、权限
- **会话数据**：会话记录、对话历史
- **消息数据**：消息内容、发送时间
- **Token账单**：Token消耗记录、费用统计

### 设计考量
- 使用连接池管理数据库连接
- 响应式环境使用R2DBC而非JDBC
- 数据库索引优化

## Redis

### 存储内容
- **Session**：用户会话信息
- **Task状态**：任务状态管理
- **限流计数**：限流计数器
- **缓存数据**：高频访问数据

### 设计考量
- 使用ReactiveRedisTemplate
- 设置合理的过期时间
- 分布式锁实现

## Vector Database

### 存储内容
- **Embedding数据**：文档向量化结果
- **知识库**：RAG检索的知识文档

### 支持的数据库
- **Milvus**：专业向量数据库
- **pgvector**：PostgreSQL向量扩展
- **Elasticsearch Vector**：全文+向量检索

### 设计考量
- 根据数据规模选择合适的向量数据库
- 建立有效的索引
- 定期数据清理

## 数据流转

```
用户请求 → MySQL（用户/会话）→ Redis（Session/限流）
            → Vector DB（Embedding/知识）→ 返回结果
```

## 代码片段

### 响应式Redis使用

```java
// 查询缓存
reactiveRedisTemplate.opsForValue().get(key)
    .switchIfEmpty(
        // 缓存未命中，从数据库查询
        webClient.get().uri("/biz/user/{id}", id)
            .retrieve().bodyToMono(UserDTO.class)
            .flatMap(user -> reactiveRedisTemplate.opsForValue()
                .set(key, user, Duration.ofMinutes(10))
                .thenReturn(user)
            )
    );
```

## 设计考量

- 冷热数据分离：热数据放Redis，冷数据放MySQL
- 向量数据库选择需要考虑数据规模和查询性能
- 响应式环境需要使用响应式数据库客户端

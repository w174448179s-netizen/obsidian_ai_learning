# 四、路由与转发 Routing

## 一句话职责
根据请求特征，将请求路由到正确的处理节点。

## 路由职责

- URL匹配：根据路径选择路由
- 服务发现：从注册中心获取服务实例
- 请求转发：将请求转发到目标服务

## 路由方式

### Path路由
- `/chat`：对话接口
- `/agent`：Agent接口
- `/v1/chat/completions`：OpenAI兼容接口

### Header路由
- 根据请求头中的模型ID选择路由
- `X-Logic-Model-Id: qwen-openai`

### Method路由
- GET：查询
- POST：创建/调用
- PUT：更新
- DELETE：删除

### 参数路由
- 根据请求参数中的模型名称路由
- 根据用户ID路由（灰度路由）

## 服务寻址

### 静态地址
- 配置文件中硬编码服务地址
- 适合开发环境或固定服务

### 注册中心
- **Nacos**：阿里出品，功能丰富
- **Eureka**：Spring Cloud原生，简单易用

### Kubernetes Service
- Service发现：通过Service名称访问
- DNS解析：Kubernetes内置DNS
- Pod负载均衡：kube-proxy自动负载均衡

## 路由策略

### 精确匹配
- 前端指定模型名 → 精确匹配路由
- 例：`model=gpt-4o` → 路由到GPT-4o适配器

### 默认降级
- 未指定模型 → 使用默认模型
- 默认模型不可用 → 路由到备用模型

### 灰度路由
- **白名单灰度**：指定用户ID列表先行体验
- **比例灰度**：按百分比随机分配流量

### 按能力路由
- 多模态请求 → 多模态模型
- 长文本请求 → 支持长上下文模型

## 路由+熔断联动

- 模型熔断后自动标记为不健康
- 路由时跳过已熔断的模型
- 自动切换到备用模型

## 网关路由配置示例

### 主线同协议路由

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: main-openai-route
          uri: lb://ai-agent-service
          predicates:
            - Path=/v1/chat/completions
            - Header=X-Logic-Model-Id, qwen-openai|deepseek-openai
          filters:
            - name: RequestRateLimiter
              args:
                redis-rate-limiter.replenishRate: 100
                redis-rate-limiter.burstCapacity: 200
            - name: CircuitBreaker
              args:
                name: mainModelCircuitBreaker
                fallbackUri: forward:/fallback/same-protocol-backup
```

### 异构备用路由

```yaml
- id: hetero-backup-route
  uri: lb://ai-hetero-sidecar
  predicates:
    - Path=/hetero/v1/chat/completions
    - Header=X-Logic-Model-Id, legacy-model|gpt-original
  filters:
    - name: HeteroModelParamFilter
    - name: RequestRateLimiter
      args:
        redis-rate-limiter.replenishRate: 10
        redis-rate-limiter.burstCapacity: 20
```

## 代码片段

### ModelRouter 接口

```java
public interface ModelRouter {
    ModelAdapter route(String modelName);            // 精确匹配
    ModelAdapter routeWithGray(ModelRequest req);    // 灰度路由
    void markUnhealthy(ModelType type);              // 熔断反馈
}
```

## 设计考量

- 路由策略可配置，支持动态刷新
- 灰度路由中白名单用户优先级高于比例灰度
- 路由+熔断联动实现自动故障转移

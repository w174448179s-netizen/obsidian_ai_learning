## 架构说明

### 核心设计理念

系统采用**分层架构**和**管道模式**设计，将AI应用的核心处理流程分解为线性流转的模块，同时通过共享上下文层和横切关注点提供统一支持。

### 数据流说明

```
用户请求 → 接入网关 → 前置过滤链 → 语义缓存层
                                    ↓
                              缓存命中?
                              ├── 是 → 直接返回结果
                              └── 否 → 意图识别子管道
                                    ↓
                              动态路由 → 业务执行域
                                    ↓
                              大模型推理 → 后置处理 → 返回结果
```

## 核心处理管道

### 1. 接入网关
- **职责**：统一API入口，屏蔽多模型厂商差异
- **核心能力**：API Key管理、认证权限校验、限流配额控制、熔断降级兜底、多模型厂商统一代理
- **详细笔记**：[[1-接入网关]]

### 2. 前置过滤链
- **职责**：输入合规检查，防止恶意输入
- **核心能力**：敏感词过滤、提示注入防御、输入合规校验、多模态识别路由
- **详细笔记**：[[2-前置过滤链]]

### 语义缓存层
- **职责**：高频问题缓存，降低模型调用成本
- **核心能力**：高频问题缓存、缓存策略配置、缓存命中判定
- **缓存机制**：命中直接返回，未命中继续下游处理

### 3. 意图识别（子管道）
- **职责**：理解用户意图，为后续路由提供依据
- **处理流程**：
  - 粗意图识别 → 判断请求类型（闲聊/问答/查询/任务）
  - 上下文加载 → 从记忆管理读取历史对话
  - 精意图识别 → 细粒度意图分类和实体抽取
  - 意图分发 → 将意图传递给动态路由
- **详细笔记**：[[3-意图识别]]

### 4. 动态路由
- **职责**：根据意图将请求分发到不同的处理路径
- **路由策略**：
  - 💬 闲聊对话 → 直接透传，组装对话历史
  - 📚 RAG知识检索 → 触发向量检索管线
  - 🔧 工具调用 → 匹配Function，注入工具定义
  - 🤖 Agent复杂任务 → 启动规划-执行-反思循环
- **详细笔记**：[[4-动态路由]]

### 5. 大模型推理
- **职责**：调用大模型生成响应
- **核心能力**：Prompt拼装、流式调用、结构化输出、多模型适配
- **详细笔记**：[[5-大模型推理]]

### 6. 后置处理
- **职责**：对模型输出进行标准化处理
- **处理流程**：
  - 脱敏 → 移除敏感信息（手机号/身份证等）
  - 格式化 → 输出格式标准化
  - 状态更新 → 更新会话状态和记忆
  - 反馈采集 → 收集用户评价（赞/踩/纠错）
- **详细笔记**：[[6-后置处理]]

## 共享上下文层

### 7. 记忆管理
- **职责**：为整个处理管道提供记忆支持
- **组成部分**：
  - 短期记忆 → 对话历史、滑动窗口、摘要压缩
  - 长期记忆 → 用户画像、偏好、历史决策
  - 会话管理 → 会话创建、恢复、过期清理
- **交互关系**：
  - 意图识别（上下文加载）读取记忆
  - 后置处理（状态更新）写入记忆
- **详细笔记**：[[7-记忆管理]]

## 全链路横切面

横切面模块贯穿整个处理管道，为所有模块提供统一服务：

### 8. 审计日志
- **职责**：全链路轨迹记录，满足合规要求
- **覆盖范围**：接入网关、前置过滤链、意图识别、大模型推理、后置处理
- **详细笔记**：[[8-审计日志]]

### 9. 可观测性
- **职责**：监控系统运行状态，追踪关键指标
- **核心能力**：延迟监控、Token用量统计、成功率追踪、埋点分析
- **覆盖范围**：接入网关、语义缓存层、大模型推理
- **详细笔记**：[[9-可观测性]]

### 10. 安全合规
- **职责**：保障输入输出双向安全
- **覆盖范围**：前置过滤链（输入安全）、后置处理（输出安全）
- **详细笔记**：[[10-安全合规]]

### 11. 异常降级
- **职责**：全局异常捕获和弹性兜底
- **覆盖范围**：核心处理管道所有模块
- **详细笔记**：[[11-异常降级]]

### 12. 效果评估
- **职责**：评估系统输出质量，持续优化
- **核心能力**：在线反馈收集、离线回归测试、版本对比分析
- **覆盖范围**：后置处理（反馈采集）
- **详细笔记**：[[12-效果评估]]
